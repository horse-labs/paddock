01

Context dies between sessions.

AI-assisted work generates a lot of context that evaporates: decisions, the roadmap, the why. Drive and Notion go stale, and the agent can't read or write them natively. Next session, you re-explain everything — or the agent guesses.

02

Memory bank — and beyond

The category has a name: memory bank — markdown kept in the repo, fed to the agent at the start of every session to restore context. Paddock is that at its core. But it goes further, adding the engineering-governance layer most memory banks lack:

Evidence-linked roadmap

Not just "what the project is" — what's done / pending, each item linked to a PR/commit/ADR. The memory bank becomes the source of truth for progress.

Delivery guardrails

A short session ritual + Definition of Ready / Done + a trunk-based git stance. Not only context — process.

Multi-repo + refinery

Many repos under one workspace, plus a raw _knowledge/ intake distilled into structure. Most memory banks are single-repo and context-only.

Cross-agent

AGENTS.md read by Copilot, Cursor, Codex and Claude. Not tied to one tool — the core is markdown + git.

03

One workspace. The agent sees everything.

Paddock isn't an island — it lives at the root of your workspace, next to the code repos and the raw project material. The agent runs from the root and cross-references memory + code + raw input at once.

Paddock acts as a refinery: it distills the raw pile in _knowledge/ into structure — specs, decisions, status. From chaos to traceable governance.

04

Adopt it in minutes.

  1. 1

    Use the template

    Click Use this template on GitHub (or the button below). You get the whole scaffold.

  2. 2

    Run setup

    Applies your project name, generates the cross-agent AGENTS.md, validates the skeleton. (Windows: Git Bash/WSL.)

    git clone <your-repo> && cd <your-repo>
bash setup.sh "My Project"
  3. 3

    Track work with evidence

    One line per item in status.md. Every ✅/🟡 carries a link — no evidence, no status.

    | E1 | Auth + sessions | ✅ | api#42 | DoD met |
  4. 4

    See the panorama

    A visual status report on demand — and an integrity lint that fails on stale markers.

    $ bash tools/report.sh
