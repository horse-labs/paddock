# plans/

Planos de implementação task-by-task (derivados de uma spec ou requisito). Um arquivo por plano, datado:
`AAAA-MM-DD-<slug>.md`.

Conteúdo típico: objetivo, arquivos a tocar, passos com checkbox, comandos de teste, verificação.
Útil pra executar em sessões separadas (humano ou agente) com checkpoints.
